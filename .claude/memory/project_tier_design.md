## 產品分層（已確認）

### 免費版
- 來源：flat table 只（第一行 header，每行一筆記錄）
- 目標：系統內建政府模板（如薪資印領清冊、各類扣繳憑單）
- 分頁：只讀單一分頁
- 檔案格式：.xlsx / .csv

### 付費版（在免費版之上擴充）
- 來源：報表型 Excel（多行/人、項目縱排等非 flat 結構）
- 目標：用戶上傳任意模板，自行框選格子標記資料槽位（A 方案）
- 分頁：多分頁全讀，支援跨分頁映射（來源A分頁 → 目標甲+乙分頁）
- 檔案格式：新增 .xls (BIFF8) 支援（透過 SheetJS 或後端）
- .xls 支援要寫成獨立的 fileParser 工廠模組，方便未來切換後端實作

**Why:** 目標用戶台灣會計事務所，輸出格式固定（政府範本），輸入格式千變萬化。
免費版解決多數 flat table 場景，付費版解決報表型與複雜模板填入。

**How to apply:** 設計新功能時先確認是免費還是付費 scope，付費功能留介面入口但 gate 起來。

## 模板填入模式（Template Fill Mode）— 已確認細節

### 操作流程
1. 上傳商家本月薪資（來源 flat table）
2. 上傳當年薪資清冊（可能已有前幾月資料）
3. 選擇/套用已存的對應設定（含月份欄設定）
4. 工具偵測模板哪些月份欄已有資料、哪些空白
5. 用戶確認「這次填哪些月」
6. 只動選定欄位，其餘完全不變（non-destructive）
7. 下載

### 狀態管理
- Excel 檔案本身就是狀態載體，用戶自己保管（下載→存本機→下月再上傳）
- 工具不存模板，只處理當次變更

### 月份對應
- 來源必須有「月份欄」（值為 1~12 或其他格式）
- 免費版：用戶手動指定「哪欄是月份」+「月份值如何對應模板月份欄」
- 付費版（AI）：自動辨識月份欄與對應關係

### 免費 vs 付費新增項目
| 功能 | 免費 | 付費 |
|------|------|------|
| 來源結構 | flat table | 報表型（多行/人） |
| 目標模板 | 系統內建政府模板 | 用戶自訂模板（框選資料槽） |
| 分頁支援 | 單一分頁 | 多分頁跨分頁映射 |
| 檔案格式 | .xlsx / .csv | 新增 .xls |
| 月份辨識 | 手動設定 | AI 自動辨識 |
| 對應設定儲存 | ✓（兩種模式都有） | ✓ |
| 多分頁範例/說明頁過濾 | 提示讓用戶決定 | 同左 |
