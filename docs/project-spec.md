## 四、功能 1：欄位對應轉換（已完成）

**用途**：把來源 Excel 的欄位名稱對應到目標格式，輸出欄位名稱重組後的 Excel。

**流程**：
```
上傳來源 Excel + 目標格式範本
  ↓
設定欄位（ExcelImportConfig：選 sheet、標記標題列）
  ↓
MappingCanvas（拖曳連線建立欄位對應）
  ↓
下載輸出 Excel
```

**已完成功能**：
- 左右拖曳上傳（支援 .xlsx / .xls / .csv）
- SheetJS 解析欄位標題 + 資料列
- React Flow 畫布，拖曳連線，連線計數 badge
- 點擊連線刪除，已連線 node 變色
- 匯出「轉換結果.xlsx」

## 五、功能 2：通用範本填入（待開發，第 1 優先）

**用途**：把任意來源 Excel 的資料，填入任意固定格式的目標 Excel 範本，保留範本的原始格式（顏色、樣式、公式）。

### 核心設計原則
- **非破壞性填入**：只覆蓋使用者指定的填入區，其餘格子（包含之前填入的資料）原封不動
- **格式保留**：讀取原始 workbook → 只寫入數值 → 輸出，樣式自然保留
- **增量填入**：同一份輸出 Excel 可分批填入（例如先填 1-6 月，再填 7-12 月）

### 流程（4 步驟）

```
步驟 1：設定來源結構
  上傳來源 Excel → HTML table 渲染 → 逐一定義欄位：「標題範圍 ↔ 資料範圍」成對新增
  （多欄標題自動逐欄拆成多個配對欄位；各欄位的資料依序位對齊成資料列）

步驟 2：設定目標結構
  上傳目標範本 → HTML table 渲染 → 定義填入區：「標題範圍（選填）↔ 填入範圍」成對新增
  （框範本上的標題格自動帶出填入區名稱，可修改；多欄標題自動逐欄拆成多個填入區）

步驟 3：對應規則設定
  左邊：來源欄位
  右邊：目標填入區
  建立對應關係 + 設定篩選條件

步驟 4：預覽 + 匯出
  預覽筆數可設定（預設 3 筆，最多 50 筆）
  確認無誤 → 匯出 / 儲存為模板
```

### 支援的檔案格式（2026-06-12 增補）
- 來源：`.xlsx` / `.xls` / `.csv`（.xls 與 .csv 經 SheetJS 轉換，只取值與合併儲存格）
- 目標範本：`.xlsx`（完整保留樣式）／`.xls`（可用但無法保留樣式，UI 顯示警告建議另存 .xlsx）

### Excel 表格渲染（HTML table）
- 把 Excel sheet 的內容渲染成 HTML table（不需完美還原格式）
- 支援基本的合併儲存格顯示
- 為未來 Excel-like 編輯功能打好基礎

### 儲存格選取 UI
- **框選只是輔助輸入**（2026-06-12 確立）：表格框選會把座標填進文字輸入框，
  輸入框可直接手動輸入／修改座標（例：`A2:A500`）—— 不受表格顯示列數上限（100 列）影響
- **欄位定義配對**（2026-06-12 增補）：來源結構 = 欄位清單，每個欄位 =「標題範圍 ↔ 資料範圍」一組。
  多欄標題（`A1:C1`）+ 多欄資料（`A2:C500`）自動逐欄拆成多個配對欄位；
  各欄位的資料範圍互相獨立，依序位對齊組成資料列
- **拖曳框選**：按住滑鼠左鍵拖曳選取矩形範圍
- **Shift + 點擊**：延伸選取範圍
- **整欄／整列選取**（2026-06-12 增補）：點欄字母選整欄（`A:F`）、點列號選整列（`2:5`），拖曳可選多欄／多列
- **開放範圍動態延伸**（2026-06-12 增補）：整欄（`A:A`）資料範圍依「各工作表實際資料列數」自動延伸，
  並自動排除該欄位的標題列 —— 解決各工作表資料筆數不一致的問題
- **半開放範圍**（2026-06-12 增補）：`B3:B` = 從 B3 到 B 欄資料底端（起點固定、不含標題列的快速選法）
- 選取中的格子高亮顯示
- 未來可擴充：Ctrl + 點擊不連續選取

### 篩選條件規則（A + C 模式）

**A — 欄位等值條件**（來源同一 sheet，不同月份在不同列）：
```
來源欄位「月份」= 1  →  填入目標範圍 B4:B15
來源欄位「月份」= 2  →  填入目標範圍 D4:D15
```

**C — 來源 sheet 名稱條件**（來源不同 sheet 對應不同月份）：
```
來源 sheet「1月」  →  填入目標範圍 B4:B15
來源 sheet「2月」  →  填入目標範圍 D4:D15
```

兩種模式可混用。未來擴充：AND 多條件組合。

### 預覽
- 顯示目標 Excel 的 HTML table，套入前 N 筆來源資料
- N 可設定（預設 3，最多 50）
- 視覺化標示：已填入的格子 vs 維持原樣的格子

### 舊版薪資清冊功能
- 新版通用範本填入上線後，首頁卡片改連結到新版
- 舊版 `TemplateFillPage` 暫時隱藏（路由保留，不刪程式碼）
- 確認新版穩定後再決定是否移除舊版程式碼

## 六、功能 3：模板儲存與套用（待開發，第 2 優先）

**用途**：儲存一整組設定，下次只需上傳資料即可套用，不需重新選格子。

### 模板儲存的內容
```json
{
  "name": "薪資清冊 2026",
  "version": 1,
  "sourceStructure": {
    "sheetSelection": "...",
    "headerRowIndex": 0,
    "dataRange": "A2:Z100"
  },
  "targetStructure": {
    "fillZones": [
      { "name": "姓名", "range": "B4:B15" },
      { "name": "薪資淨額", "range": "C4:C15" }
    ]
  },
  "mappingRules": [
    {
      "sourceField": "姓名",
      "targetZone": "姓名",
      "condition": { "type": "column-value", "column": "月份", "value": 1 }
    }
  ]
}
```

### 儲存機制
- **本機儲存**：localStorage（免費功能）
- **匯出 JSON**：下載備份，跨裝置使用
- **匯入 JSON**：還原模板
- **未來付費功能**：雲端同步（需後端，不在 MVP 範圍）

### 套用模板流程
```
選擇模板（首頁 or Header 入口）
  ↓
上傳來源資料（新鮮資料）
  ↓
自動套用已儲存的結構定義 + 對應規則
  ↓
跳至預覽步驟（可修改設定）
  ↓
匯出
```

### 儲存時機
- 在**預覽頁**底部提供「儲存為模板」按鈕（確認設定正確後再存）
- 輸入名稱 → 存入 localStorage

### 功能 1 的模板儲存
- 欄位對應轉換也支援儲存模板（儲存欄位對應關係）
- 套用後直接進 MappingCanvas，已預填連線

## 七、功能 4：AI 欄位建議（待開發，最後優先）

**用途**：自動預填欄位對應連線，減少手動拖曳工作量。

### 範圍（MVP）
- **只支援功能 1（欄位對應轉換）**
- 範本填入的 AI 建議複雜度高，等欄位轉換穩定後再延伸

### 設計

| 項目 | 決策 |
|------|------|
| Key 儲存 | `sessionStorage`（關分頁即清） |
| AI Provider | Gemini `gemini-3.5-flash`，工廠模式（可擴充 Claude / OpenAI） |
| UI 入口 | Header 右上角齒輪 → 設定 Modal |
| 無 Key 狀態 | AI 建議按鈕仍顯示，點了開設定 Modal 引導填寫 |
| 回傳格式 | `[{"from":"...","to":"...","confidence":0.92}]` |
| 信心分數顯示 | ≥0.8 藍實線，0.5–0.8 橘虛線，<0.5 不畫線 |
| Loading | 按鈕 spinner + 畫布半透明遮罩 |
| 錯誤處理 | Toast 通知，不動現有連線；Key 無效 → 清除 + 提示重設 |
| 商業化路徑 | 上後端後改用平台 key，BYOK 整個移除 |
